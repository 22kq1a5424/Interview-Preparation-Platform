Explain Object-Oriented Programming (OOP) concepts. ▼

OOP is a programming paradigm based on the concept of "objects," which can contain data and code. Key concepts include:

  • Encapsulation: Bundling data and methods that operate on the data within a single unit (class/object), hiding internal details.
  • Inheritance: A mechanism where a new class (subclass) inherits properties and behaviors from an existing class (superclass).
  • Polymorphism: The ability of an object to take on many forms, often meaning a single interface can be used for different data types.
  • Abstraction: Hiding complex implementation details and showing only the essential features of an object.

What is the difference between a `stack` and a `queue`? ▼

Both are linear data structures, but they differ in how elements are added and removed:

  • Stack (LIFO - Last In, First Out): Elements are added and removed from the same end, called the "top." Think of a stack of plates. The last plate added is the first one removed. Operations: Push (add), Pop (remove), Peek (view top).
  • Queue (FIFO - First In, First Out): Elements are added at one end (rear/tail) and removed from the other end (front/head). Think of a line of people. The first person in line is the first one served. Operations: Enqueue (add), Dequeue (remove), Front/Peek (view front).

What is a database index and why is it used? ▼

A database index is a data structure (like a B-tree or hash table) that improves the speed of data retrieval operations on a database table. It's similar to the index in a book.

Why it's used:

  • Faster Queries: Instead of scanning the entire table (full table scan), the database can quickly locate the required data using the index.
  • Improved Performance: Speeds up `SELECT`, `WHERE`, `JOIN`, and `ORDER BY` clauses.
  • Uniqueness: Can enforce uniqueness constraints on columns.
However, indexes come with overhead: they require storage space and can slow down data modification operations (`INSERT`, `UPDATE`, `DELETE`) because the index also needs to be updated.

Explain the concept of 'concurrency' in programming. ▼

Concurrency in programming refers to the ability of different parts of a program or system to execute independently or out of order. It's about dealing with many things at once.

Key aspects:

  • Threads/Processes: Programs can be broken down into multiple threads or processes that run seemingly simultaneously.
  • Parallelism vs. Concurrency:
    • Concurrency: Deals with *many tasks at once* (even if switching between them on a single core).
    • Parallelism: Deals with *many tasks simultaneously* (requires multiple cores/processors).
  • Challenges: Concurrency introduces challenges like race conditions, deadlocks, and starvation, requiring synchronization mechanisms (locks, semaphores, mutexes) to ensure data integrity and correct execution.

What is REST API? ▼

REST (Representational State Transfer) is an architectural style for designing networked applications. REST APIs are a way for two computer systems to communicate over HTTP in a stateless manner.

Key principles of REST:

  • Stateless: Each request from client to server must contain all the information needed to understand the request. The server doesn't store client context between requests.
  • Client-Server: Clear separation of concerns between the client and server.
  • Cacheable: Responses should explicitly or implicitly define themselves as cacheable to prevent clients from requesting the same resource multiple times.
  • Uniform Interface: A consistent way to interact with resources (e.g., using standard HTTP methods like GET, POST, PUT, DELETE).
  • Layered System: Intermediaries (proxies, load balancers) can be inserted between client and server without affecting their interactions.
REST APIs typically use JSON or XML for data exchange.

How does HTTPS work? ▼

HTTPS (Hypertext Transfer Protocol Secure) is the secure version of HTTP, using SSL/TLS encryption. It ensures data transferred between your browser and a website is encrypted and authenticated.

Key steps:

  • SSL/TLS Handshake: When you connect to an HTTPS website, a handshake occurs:
    • Your browser sends a "Client Hello" with supported SSL/TLS versions and cipher suites.
    • The server responds with a "Server Hello," selecting a version and cipher suite, and sends its SSL certificate.
    • Your browser verifies the certificate's authenticity (checking the Certificate Authority, expiration, domain match).
    • If valid, the browser generates a pre-master secret, encrypts it with the server's public key from the certificate, and sends it to the server.
    • Both client and server then use this pre-master secret to generate a symmetric session key.
  • Symmetric Encryption: All subsequent communication during the session is encrypted using this fast symmetric session key. This protects data from eavesdropping and tampering.
  • Authentication: The SSL certificate verifies the server's identity, preventing man-in-the-middle attacks.
